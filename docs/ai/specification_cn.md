# 面向 AI 的文档撰写规范 (AI-Oriented Documentation Specification)

## 核心原则

1. **高信息密度**：避免冗长的解释，优先使用代码、表格、列表和形式化描述。
2. **结构化呈现**：文档应易于被 AI 解析。使用明确的分级标题、标准的 Markdown 语法（如表格、代码块）。
3. **闭环说明**：任何功能点必须包含“声明/定义”与“用法示例”。
4. **零歧义**：明确指出可选性、默认值、限制条件和副作用。

---

## 撰写规范

### 1. 模块/宏/函数概览
每个主要单元应首先提供一句话定义的摘要，随后列出其核心能力。

### 2. 声明与参数 (Declarations)
宏或复杂函数应给出形式化的语法结构或类型声明。

**格式要求：**
- 使用代码块列出声明。
- 参数说明应包含：名称、类型、是否可选、默认值、用途。

### 3. 用法示例 (Usage)
每个功能点必须配有最短可行代码示例（Minimal Working Example）。

**格式要求：**
- 区分成功路径与错误路径。
- 标注生成的代码（对于宏而言）。

### 4. 属性与元信息 (Attributes & Metadata)
对于过程宏，需详细列出所有支持的属性（Attributes）。

**表格格式：**
| 属性 | 位置 | 参数 | 说明 |
| :--- | :--- | :--- | :--- |
| `#[display]` | 变体 | `"template"` | 设置渲染模板 |

### 5. 高阶模式 (Advanced Patterns)
列出非显而易见但极具价值的使用组合、泛型约束或与其他库的协作。

---

## 示例模板 (AI-Reference Format)

### `example_macro!` (Macro)
**声明：**
```rust
macro_rules! example {
    ($name:ident = { ... }) => { ... }
}
```

**支持属性：**
- `#[attr1(arg)]`: 作用于...

**用法：**
```rust
example! { Name = { ... } }
```
